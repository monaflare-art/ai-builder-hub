# AI Builder Hub Content Style Guide V2

This guide is based on the local benchmark set in `docs/content-benchmarks/`. The benchmark files are screenshot-style PDFs from high-conversion Chinese tool, VPS, domain, SEO, Shopify, and affiliate articles. Most PDFs do not expose selectable text, so the learning pass focused on visible layout, page flow, screenshot density, table placement, CTA rhythm, and trust signals.

## Shared Benchmark Patterns

### Title Structure

- Use direct search-intent titles: `Tool Review 2026`, `Tool A vs Tool B`, `Best X for Y`, `How to X`, and `X Pricing / Configuration Guide`.
- Put the product, category, or concrete outcome at the front of the title.
- Add year only when freshness matters: reviews, pricing, domain registrar comparisons, hosting recommendations, and VPS plans.
- Avoid clever editorial titles. The benchmark titles are clear before they are stylish.

### Opening Hook

- Start with the reader's decision problem, not a broad industry intro.
- Give a quick conclusion early: who should use the tool, who should avoid it, and what the article will prove.
- Add credibility fast: personal test notes, account screenshots, control panel screenshots, pricing screenshots, or real configuration evidence.
- Keep the opening practical. No generic AI SEO paragraphs.

### Product Recommendation Order

- Start with the reader's use case, then the product fit, then alternatives.
- Do not push affiliate links first. Benchmarks introduce the product, show evidence, compare details, and then present the link.
- For best-tools posts, lead with a quick table, then explain the top picks one by one.
- For reviews, lead with verdict and best-fit cases, then pricing, setup, screenshots, pros, cons, and alternatives.

### Image and Screenshot Use

- Use real screenshots as evidence, not decoration.
- Strong screenshot types: dashboard, pricing page, order page, server region list, DNS panel, control panel, keyword report, Shopify admin, affiliate dashboard, and support/documentation page.
- Place screenshots immediately after the claim they prove.
- Use captions that state what the screenshot proves.
- Do not rely only on abstract cover art for review content. Abstract SVGs are acceptable for index cards and generic guides, but review pages need real proof.

### Tables

- Use tables near the top to compress decisions.
- Common benchmark tables:
  - quick verdict table
  - pricing/configuration table
  - best-for/not-best-for table
  - feature comparison table
  - plan comparison table
  - alternatives table
- Table cells should contain judgments, not filler.
- Good columns: `Best for`, `Not best for`, `Starting point`, `Beginner risk`, `Evidence needed`, `Alternative`.

### Affiliate Link Placement

- Put affiliate links after value has been established.
- Recommended positions:
  - after quick verdict
  - after pricing/configuration explanation
  - after setup walkthrough
  - after final recommendation
- Always include disclosure near commercial CTAs.
- If no approved affiliate URL exists, link only to official website or internal detail page.
- Never write "official partner" unless legally true.

### FAQ

- FAQs answer objections, not generic definitions.
- Use 4-6 questions per commercial article.
- Good FAQ topics: refund, renewal price, data center choice, beginner suitability, migration, support, affiliate disclosure, and when to choose alternatives.
- Answers should be short, specific, and decision-oriented.

### Closing CTA

- End with a practical next step.
- For reviews: `Visit official website`, `Compare alternatives`, `Read disclosure`.
- For best-tools posts: `Start with the recommended stack`, then link to 2-3 relevant tool pages.
- For comparisons: summarize who should choose each option before showing CTAs.
- Do not place a block of unrelated Recommended Tools only at the bottom. Commercial recommendations must appear naturally throughout the article.

## Tool Review Template

1. H1: `{Tool} Review 2026: Is It Worth It for {Audience}?`
2. Hero image or real product screenshot.
3. Quick verdict box:
   - best for
   - not best for
   - pricing summary
   - affiliate status
   - evidence status
4. Short disclosure.
5. Overview: what the tool does and why readers compare it.
6. Pricing / plan table.
7. Setup walkthrough with screenshots.
8. Pros with evidence.
9. Cons with limitations.
10. Alternatives table.
11. Who should choose it.
12. FAQ.
13. Final CTA.

## Best Tools Template

1. H1: `Best {Category} Tools for {Audience} in 2026`
2. Short hook that names the decision problem.
3. Selection criteria.
4. Quick comparison table with 5-10 tools.
5. Top 3 recommended stack.
6. Individual tool sections:
   - short verdict
   - best for
   - pricing summary
   - evidence screenshot
   - internal tool page CTA
7. Avoid-this-if section.
8. FAQ.
9. CTA to tools directory and disclosure.

## Comparison Template

1. H1: `{Tool A} vs {Tool B}: Which Should {Audience} Choose?`
2. One-paragraph verdict.
3. Quick comparison table.
4. Use-case split:
   - choose A if
   - choose B if
   - choose neither if
5. Pricing/configuration comparison.
6. Setup and beginner-risk comparison.
7. Evidence screenshots from both tools.
8. Alternatives.
9. FAQ.
10. Final CTA with both options represented fairly.

## Real Screenshot and Evidence Layer

Every commercial review should include an evidence layer before strong recommendations.

Required evidence fields:
- pricing screenshot needed
- dashboard/control panel screenshot needed
- setup screenshot needed
- account or order flow screenshot needed when relevant
- real test notes needed for VPS/hosting performance claims
- renewal price evidence needed for hosting/domain reviews

Evidence writing rules:
- Say what was tested and what was not tested.
- Distinguish public information from hands-on evidence.
- Do not claim performance, support quality, or approval status without proof.
- When evidence is missing, mark it as needed and keep the recommendation cautious.

## Prohibited Patterns

- Do not write empty AI SEO intros.
- Do not use vague phrases like "in today's digital age" or "revolutionize your workflow".
- Do not only put Recommended Tools at the bottom.
- Do not invent affiliate approval.
- Do not write "official partner" without proof.
- Do not invent discounts, coupons, or renewal prices.
- Do not compare tools only by homepage claims.
- Do not publish review conclusions without evidence fields.
- Do not use screenshots as decoration without explaining what they prove.
