# Affiliate Money Page Template

Purpose: reusable structure for future high-intent review, comparison, and best-tools pages. This is a framework only; do not rewrite articles until a dedicated rewrite pass.

## Page Metadata

- Primary keyword:
- Secondary keywords:
- Tool/category:
- Search intent:
- Affiliate status:
- Evidence status:
- Last checked:
- Target CTA:

## 1. Quick Verdict

Place immediately after the intro or hero.

Required fields:

- Best choice for:
- Avoid if:
- Pricing snapshot:
- Best alternative:
- Affiliate status:
- Evidence status:

Example layout:

| Verdict item | Answer |
| --- | --- |
| Best for |  |
| Avoid if |  |
| Starting price |  |
| Main alternative |  |
| Evidence checked |  |

CTA location: one primary CTA below the verdict. If affiliate is not approved, link to official site or internal tool page only.

## 2. Best For

Use 3-5 bullets. Each bullet must name a concrete user or use case.

Good examples:

- Beginners launching a WordPress affiliate site.
- Developers who need Docker and long-running workers.
- Founders buying a domain before connecting Vercel.

## 3. Avoid If

Use 3-5 bullets. This increases trust and reduces low-quality clicks.

Good examples:

- You only need a static landing page.
- You do not want to manage Linux updates.
- You have not chosen a niche or content plan yet.

## 4. Pricing Summary

Use one pricing table near the top third of the page.

Evidence layer location: include screenshot or note immediately after the table.

Table columns:

| Plan / item | Current public price | Renewal or hidden cost | Best fit | Evidence needed |
| --- | --- | --- | --- | --- |

Rules:

- Do not invent discounts.
- Do not state exact renewal prices without evidence.
- Add `Last checked` date when using live pricing.

## 5. Key Features

Use features only when they affect buying decisions.

Recommended structure:

| Feature | Why it matters | Beginner risk | Screenshot |
| --- | --- | --- | --- |

Screenshot location: after the feature table, add dashboard/control-panel screenshots that prove the claims.

## 6. Real-World Use Cases

Write 3-4 practical scenarios.

Examples:

- Launching an affiliate site.
- Deploying a Next.js app.
- Buying and connecting a domain.
- Running a small VPS worker.
- Starting an ecommerce test store.

Each use case should end with a next-step link to a relevant tool page or supporting article.

## 7. Builder Notes

This section should feel like hands-on experience, not vendor copy.

Include:

- Setup friction.
- Dashboard clarity.
- Billing or renewal concerns.
- Common beginner mistake.
- What still needs testing.

Evidence layer location: place screenshots in this section if the article discusses setup, dashboard, DNS, server creation, or app deployment.

## 8. Pros

Use evidence-backed pros.

Format:

- Pro: short claim.
- Why it matters: one sentence.
- Evidence: screenshot/test note/link placeholder.

Avoid generic pros like "easy to use" unless backed by setup evidence.

## 9. Cons

Cons should be specific and useful.

Include:

- Cost/renewal caveats.
- Technical skill required.
- Migration risk.
- Feature limits.
- When another tool is better.

## 10. Alternatives

Every money page needs alternatives above the FAQ, not only at the bottom.

Comparison table:

| Alternative | Best for | Not best for | Link |
| --- | --- | --- | --- |

For review pages, include 3-5 alternatives.

For comparison pages, include a "choose neither if" row.

## 11. FAQ

Use 4-6 questions. Keep answers direct and decision-focused.

Recommended FAQ patterns:

- Is this tool good for beginners?
- What does it cost after renewal?
- Can I connect a custom domain?
- Is it better than [main alternative]?
- Do I need this if I already use [related tool]?
- Does AI Builder Hub use affiliate links?

## 12. Final Verdict

Final verdict should summarize:

- Who should choose it.
- Who should avoid it.
- Best alternative.
- What evidence is still missing.

Do not introduce a new product recommendation in the final verdict.

## 13. Affiliate CTA Block

Place after final verdict.

Required elements:

- Primary button: `Visit Official Website` or affiliate destination if approved.
- Secondary button: `Compare Alternatives` or `Read Disclosure`.
- Disclosure text: "Some links may be affiliate links. We may earn a commission at no extra cost to you."
- Status note: approved, pending, not-applied, or unavailable.

Do not write:

- "official partner"
- fake coupon language
- guaranteed results
- unsupported best-in-market claims

## Screenshot Locations

Recommended screenshot map:

1. Hero/early proof: pricing page or dashboard.
2. Pricing Summary: plan table screenshot.
3. Key Features: dashboard/control panel screenshot.
4. Builder Notes: setup or configuration screenshot.
5. Alternatives: optional comparison table screenshot if using external evidence.
6. Final CTA: no screenshot needed; keep it clean.

## Evidence Layer Locations

Evidence should appear before strong recommendations.

Place evidence:

- after Quick Verdict for high-intent review pages
- after Pricing Summary for hosting/domain/VPS pages
- inside Builder Notes for setup-heavy tools
- before Final Verdict for performance/support claims

Evidence states:

- `Verified`: screenshot/test exists.
- `Public info only`: based on public pages.
- `Needed`: claim should stay cautious.

## Comparison Table Locations

Use comparison tables:

- immediately after Quick Verdict on comparison pages
- after Pricing Summary on review pages
- before individual tool sections on best-tools pages
- before Alternatives on tutorial pages with buying intent

Core comparison columns:

| Tool | Best for | Avoid if | Pricing summary | Evidence status | CTA |
| --- | --- | --- | --- | --- | --- |
